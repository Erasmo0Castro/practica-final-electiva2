Descripción de la práctica

En esta práctica vamos a realizar un poco de todo lo aprendido en el curso de DevOps. El objetivo es crear una página web simple, configurar pruebas automáticas y desplegarla utilizando Docker, GitHub Actions y GitHub Pages.

Pasos a seguir:

  1. Crear un repositorio Git con una página web HTML de "Hola Mundo".
    • Crea un archivo `index.html` con un simple mensaje de bienvenida.
  2. Crear una prueba unitaria para la página creada.
    • Desarrolla una prueba unitaria sencilla para verificar que la página cargue correctamente.
  3. Crear un servidor web utilizando Docker (Nginx o Apache) para servir la página localmente.
    • Crea un archivo `Dockerfile` para configurar el servidor web que servirá la página.
  4. Crear el archivo YAML de GitHub Actions para ejecutar automáticamente la prueba unitaria al actualizar el repositorio.
    • Configura un flujo de trabajo en GitHub Actions para que se ejecute la prueba cada vez que subas cambios al repositorio.
  5. Subir el repositorio a GitHub y publicar la página en GitHub Pages.
    • Publica tu página en GitHub Pages para que sea accesible desde cualquier navegador.

Para más detalles, consulta la documentación oficial de GitHub Actions y GitHub Pages.
